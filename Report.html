1. Objective

Цель проекта — спроектировать и разработать простую консольную систему e‑commerce на Java с использованием принципов ООП: инкапсуляции, наследования и работы с коллекциями.

2. Requirement Analysis

  • Пользователи: Customer, Employee на основе базового класса User.
  • Товары: Book, Electronics, Clothing на основе базового класса Product.
  • Заказы: Order содержит список позиций OrderItem, связан с конкретным Customer.
  • Операции: добавление пользователей/товаров, просмотр списков, оформление и просмотр заказов.
  • Хранение данных: коллекции ArrayList в классе Store.
  • Взаимодействие: текстовые меню через Scanner.

3. Object Modeling

Отношения включают наследование (User → Customer/Employee, Product → Book/Electronics/Clothing) и ассоциации (Store агрегирует пользователей, товары и заказы; Order агрегирует OrderItem и связан с Customer).

5. Design Notes

  • Инкапсуляция: все поля приватные; предоставлены геттеры/сеттеры.
  • Наследование: выделены базовые абстракции User и Product с реализацией через подклассы.
  • Полиморфизм: переопределён метод displayInfo() в продуктах и пользователях.
  • Менеджмент ID: Store назначает ID продуктам и заказам.
  • Валидация стока: метод reduceStock() предотвращает отрицательные остатки.
  • Чеки: Order.printReceipt() выводит оформленный чек.
